# Section Acceptance Checklists

## Purpose

This file provides concise implementation-ready acceptance targets for each major section.

## Create

- editor-first layout is preserved
- source input and project context are clear
- AI companion supports editing instead of blocking it
- save draft, schedule, and publish actions are visible and unambiguous
- no credit or billing language is visible

## Calendar

- left side is visually dominated by the planner
- right side acts as the only detail companion
- no duplicate scheduling CTAs are shown
- day selection and agenda stay synchronized
- no unnecessary event popups remain

## Drafts

- drafts are easy to scan
- continue-editing is the primary action
- schedule from draft is obvious
- list is not overloaded with metrics

## Published

- published history is easy to scan
- platform and timestamp are always visible
- opening the real post is straightforward

## Failed

- failure reason is visible
- retry, reschedule, and edit-again actions are easy to understand
- layout prioritizes recovery, not raw logs

## System Analytics

- metrics summarize workflow state clearly
- charts are minimal and purposeful
- activity feed uses workflow language
- no SaaS billing concepts remain

## Integration Center

- connected accounts are easy to identify
- connect and disconnect flows are obvious
- popup blocked or OAuth failure states have a recovery path
- route usage is canonical and consistent

## Global

- route, rendered section, and sidebar highlight always match
- one dominant purpose per screen
- one primary CTA per state
- thesis-facing vocabulary is used throughout
