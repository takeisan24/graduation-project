# Project Documentation

This directory gathers the thesis-facing materials used to explain the scope,
architecture, implementation roadmap, and defense narrative of CreatorHub.

## Main Documents

- `PROJECT_TIMELINE.md`: maps the proposal schedule to the rebuilt repository
  milestones and commit groups
- `ARCHITECTURE.md`: summarizes the system structure, data flow, and module
  boundaries
- `FEATURE_MAP.md`: connects thesis requirements to implemented features
- `DEFENSE_NOTES.md`: short notes for presenting the project and answering
  common review questions
- `thesis-feature-scope.md`: scope decision document for what is emphasized,
  de-emphasized, and how the full workflow is framed for the thesis

## Detailed Blueprints

The `blueprints/` folder contains detailed functional and technical notes for
the main modules:

- create workspace
- calendar planning
- drafts and post lifecycle
- integration center
- cross-cutting architecture rules
- implementation checklists
